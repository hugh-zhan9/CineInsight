# AI 智能增强概要设计

## 设计原则
- 本地优先
- 资源优先级：CPU < 内存 < 存储
- 默认无模型下载
- 远程能力可插拔
- 所有分析结果都可审查、可撤销

## 总体思路
当前项目不复制 Immich 的完整机器学习栈，而是借它的架构思路：
- 统一入口
- 分任务分析
- 结果缓存
- 健康降级
- 审核队列

在本项目中，机器学习能力收敛为三层：
1. 低成本视频证据层：抽帧、字幕、文件名、路径、时长、分辨率
2. 轻量视觉层：人脸检测、人脸签名、重复面部聚类
3. 标签生成层：基于规则和可选远程模型生成候选标签

## 交付目标
- 视频打标可离线运行
- 低配设备可用
- 人脸能力可做检测和识别
- 远程模型失败时仍有可用结果

## 与 Immich 的差异
Immich 的 ML 服务面向多用户相册，默认按图片资产做 CLIP 向量、OCR、人脸 embedding 和 person 聚类。本项目面向本地视频库，优先需求是“视频候选标签”和“可解释的人脸线索”，因此采用更轻的路线：
- 不新增常驻 Python ML 服务
- 不默认下载 HuggingFace/InsightFace/CLIP 模型
- 先用 Go 进程内轻量检测和规则分析
- 继续通过候选审核保护标签质量

## 本轮实现边界
- 已实现远程缺失/失败时的本地标签 fallback
- 已实现人脸检测记录表和聚类表
- 已实现 Pigo 检测器接入
- 已实现人脸摘要进入本地打标证据
- 已实现轻量自然语言搜索入口，优先复用字幕、标签、人脸和元数据筛选
- 已实现前端视频列表的人脸分析入口
- 默认路径不下载模型
