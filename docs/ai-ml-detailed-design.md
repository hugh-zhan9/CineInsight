# AI 智能增强详细设计

## 模块划分

### 1. 证据采集
输入：
- 视频文件
- 字幕文本
- 抽帧图像
- 文件名与路径
- 视频元数据

输出：
- 规则证据
- 视觉签名
- 人脸检测结果
- 人脸签名

### 2. 轻量视觉分析
实现策略：
- 抽取少量低分辨率帧
- 计算低成本视觉特征
- 提取明显的场景/内容信号
- 作为标签候选的证据来源

### 3. 人脸检测与识别
实现策略：
- 使用纯 Go 级联检测做人脸定位
- 对检测到的人脸区域计算稳定签名
- 以签名距离做跨视频聚类
- 聚类结果先进入候选态，不直接写入正式标签

当前实现：
- `models.VideoFace`：记录视频帧中的人脸框、分数、签名和来源。
- `models.FaceCluster`：记录同一签名聚类。
- `services.VideoFaceService`：接收检测器结果并写入 `video_faces` / `face_clusters`。
- `services.PigoVideoFaceDetector`：使用 Pigo `facefinder` 级联文件，从少量抽帧中检测人脸。
- `services.AITaggingExtractor`：读取 `video_faces` 汇总为 `detected_face_count`。
- `services.LocalHeuristicAITaggingClient`：有人脸证据时生成“人物”候选。

### 4. 标签生成
优先级：
1. 人脸信号
2. 字幕信号
3. 文件名/路径信号
4. 视觉帧信号
5. 远程模型信号（可选）

执行策略：
- 如果远程配置缺失：直接本地分析。
- 如果远程调用失败：回退本地分析。
- 如果本地分析没有高/中置信候选：记录 skipped。
- 所有候选仍写入 `ai_tag_candidates`，用户批准后才写正式 `video_tags`。

### 5. 结果持久化
沿用现有候选审查模型：
- 生成候选
- 人工批准/拒绝
- 审批后才写正式标签

### 6. 轻量自然语言搜索
实现策略：
- 不默认建立 CLIP 向量索引，也不下载文本/图像模型。
- 先解析常见口语查询，再映射到现有关系型索引和元数据。
- 搜索入口保持分页能力，后续如接入可选语义向量索引，可替换同一后端 API 的内部实现。

当前实现：
- `services.VideoService.SearchVideosSmart`：智能搜索入口。
- `services.parseVideoSearchIntent`：解析“有人脸”“带舞台标签”“字幕里提到...”“4K”“竖屏”等轻量意图。
- `app.SearchVideosSmart`：Wails API 暴露给前端。
- 前端视频列表新增“智能搜索”模式，并继续叠加标签、体积、分辨率筛选。

## 低资源策略
- 限制抽帧数量
- 限制人脸分析帧数
- 人脸检测可选启用
- 无检测资源时自动降级
- 远程调用失败时自动降级

默认参数：
- 人脸分析抽帧数：3
- 人脸抽帧最大宽度：480
- Pigo 级联资源：约 234KB
- AI 打标抽帧最大数量：沿用现有上限 8
- 智能搜索默认不下载模型，仅使用数据库索引和已有证据表

## 风险控制
- 所有新分析必须有测试覆盖
- 任意外部依赖不可作为唯一成功路径
- 不允许自动下载大模型作为默认步骤
